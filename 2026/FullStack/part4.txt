15. Why is @Repository important beyond being a marker annotation?
---------------------------------------------------------------------------------------------
    -> Why Repository annoation is need because
       *) Without this annotation the service layer (@Service) will get the jpa,jdbc error as technology spcific errors
        which is hard to understand.
        Ex: 
        // Without @Repository, you get raw JPA exceptions
            javax.persistence.NoResultException
            javax.persistence.PersistenceException
            org.hibernate.HibernateException
            java.sql.SQLException (error code 1062 — MySQL duplicate key)
    -> By using the @Resistory annotation 
        -> Spring will register the PersistenceExceptionTranslationPostProcessor bean
        -> THis bean wrapps the @Repository to proxy.
        -> So what ever error thrown the Repository proxy will send to the PersistenceExceptionTranslator
        -> This Translator will check with DataAccessException  unchecked hiracky and sned back the clean and understandable errors to Service
        Ex:
            DataAccessException  (unchecked)
                ├── DataIntegrityViolationException   ← duplicate key, FK violation
                ├── EmptyResultDataAccessException    ← expected 1 row, got 0
                ├── QueryTimeoutException             ← slow query timeout
                ├── OptimisticLockingFailureException ← @Version conflict
                ├── DeadlockLoserDataAccessException  ← DB deadlock
                └── TransientDataAccessException      ← retry-able failures
        -> Full flow will be 
            Your Service calls Repository method
                        ↓
            Proxy intercepts the call
                        ↓
            Repository method throws HibernateException / SQLException
                        ↓
            Proxy catches it → runs PersistenceExceptionTranslator
                        ↓
            Translator maps it → Spring's DataAccessException hierarchy
                        ↓
            Your Service receives a clean, vendor-neutral exception

21. Why does Transactional sometimes not work in Spring Boot?
----------------------------------------------------------------------
    -> @Transactional fails when we bypass the spring proxy.
    -> Common issue
        ->Self invocation (Calling a @Transactional method from within the same class bypasses the proxy entirely because this refers to the real object, not the proxy)
        -> Private method 
        -> WHen class is not managed by the Spring
        ->Transaction relies on proxy interception which require public methods and external calls

------------------------------------------------------------------------------------
