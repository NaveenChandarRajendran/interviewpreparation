Kafka Basics Notes
1. Topic
What is it?

A Topic is a logical channel where producers publish messages and consumers read them.

Think of it as a folder that stores related messages.

Example
Topic: trade-created

Trade 100 Created
Trade 101 Created
Trade 102 Created

In a Spring Boot application:

kafkaTemplate.send("trade-created", tradeEvent);

The message is sent to the trade-created topic.

2. Partition
What is it?

A Partition is a physical division of a Topic.

Kafka splits a topic into multiple partitions to support parallel processing and high throughput.

Example
Topic : trade-created

├── Partition 0
├── Partition 1
└── Partition 2

Messages might be stored like this:

Partition 0
-------------
Trade100
Trade103

Partition 1
-------------
Trade101
Trade104

Partition 2
-------------
Trade102
Trade105
Why partitions?
Faster processing
Multiple consumers can work simultaneously
Better scalability
Maintains order within a partition
3. Consumer Group
What is it?

A Consumer Group is a group of consumers working together to read messages from a topic.

Kafka assigns partitions among the consumers so each message is processed only once within that group.

Example
Topic : trade-created

Partition 0
Partition 1
Partition 2

Notification Service has 3 instances.

Consumer Group : notification-group

Instance 1
Instance 2
Instance 3

Kafka assigns:

Partition 0 → Instance 1

Partition 1 → Instance 2

Partition 2 → Instance 3

Each instance processes different messages.

4. Offset
What is it?

An Offset is the position (or index) of a message inside a partition.

Every message in a partition gets a unique offset.

Example
Partition 0

Offset 0 → Trade100

Offset 1 → Trade101

Offset 2 → Trade102

Kafka uses offsets to know:

Which messages have already been consumed.
Which message should be read next.

Important: Offsets are unique only within a partition, not across the whole topic.

Interview One-Liners
Topic

A Topic is a logical channel where producers publish messages and consumers read them.

Partition

A Partition is a physical division of a Topic that enables parallel processing, scalability, and ordered message storage within that partition.

Consumer Group

A Consumer Group is a set of consumers working together. Kafka distributes partitions among them so each message is processed only once per group.

Offset

An Offset is the unique position of a message within a partition. Kafka uses it to track which messages have been consumed.
