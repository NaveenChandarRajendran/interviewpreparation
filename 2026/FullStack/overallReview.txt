Virtual thread
------------------------------------------
    -> Virtual thread introduced in java 21 are lightweight threads managed by JVM rather than OS.
    -> Before thread its traditional thread which maps 1:1 to OS thread which is expensive to scale.
    -> Virtual thread let us run millions of them with small pool of OS thread.
How it working is
    -> WHen Virtual thread block any code like (database connection,rest api call), the JVM will unmount that thread from carrier thread.
    THe carrier thread is nothing but OS thread.Real thread.
    -> JVM unmounths the thread and take new thread. when the results came from old thread the n it will remount to the carrier thread.

What is Circuit Breaker
------------------------------------------------
    -> Circuit Breaker is a fault tolorence pattern used in microservice to prevent repeeated calls to the failing services.
    -> If a service is failing for sometime , then the circuit will open as it will be wait for some time.
    -> After the wait time like 5 to 10 mins then it moves to the half open to test the servicve is up or not.
There are three states for Circuit Breaker
    -> Closed State (Everything is healthy other service are giving response)
    -> Open State (Too many failure so immediaidlty return fallback response)
    -> Half Open (After some time , Only few request will be sent to check the service is up)

-----------------------------------------------------------------------

    Ex: if two thread simultaneously tries to change the same hashmap means it may cause issue.
        The issue can be incorrect data,Lost updates etc,
    -> So to control this we will use ConcurrentHashMap.
    ConcurrentHashMap<Integer, String> users = new ConcurrentHashMap<>();
    -> Many threads can safe read and write
    -> It locks only the required position
